# Strategy Dashboard v1.0

Config-driven multi-strategy dashboard generator. Each strategy renders as an independent panel; multiple strategies display as tabbed views.

## Architecture

```

                              ‚Üì

```

## Usage

### Config-driven (recommended)

```bash
python3 generate_dashboard.py --config dashboard_config.json --out dashboard.html
```

### Legacy single-strategy

```bash
python3 generate_dashboard.py --state grid_state_v4.json --log /tmp/grid_bot_cron.log --out dashboard.html
```

### Auto-detect

Without arguments, looks for `dashboard_config.json` in the same directory, then falls back to `grid_state_v4.json`.

### Strategy Fields

| Field | Required | Description |
|-------|----------|-------------|
| `id` | Yes | Unique identifier for tab switching |
| `name` | Yes | Display name in header and tabs |
| `type` | Yes | Renderer type (currently: `grid`) |
| `state_file` | Yes | Path to strategy state JSON |
| `log_file` | No | Path to cron log file |

## Adding a New Strategy Type

1. Implement `render_<type>_strategy(strategy_cfg: dict) -> dict` in `generate_dashboard.py`
2. Return dict with keys: `id`, `name`, `version`, `status_color`, `status_label`, `summary_items`, `position_html`, `chart_svg`, `chart_footer`, `left_panel`, `right_panel`
3. Register in `STRATEGY_RENDERERS`: `"<type>": render_<type>_strategy`
4. Add entry to `dashboard_config.json`

## Deployment (VPS)

The dashboard is generated by cron alongside strategy scripts:

```bash
# In crontab (after strategy tick):
python3 ~/.openclaw/scripts/generate_dashboard.py \
    --config ~/.openclaw/scripts/dashboard_config.json \
    --out ~/.openclaw/scripts/dashboard.html
```

## Design

- OKX Onchain OS dark theme with aurora background
- Self-contained HTML (no external JS dependencies)
- Responsive layout (mobile-friendly)
- Tabbed view for multiple strategies
- Each strategy panel includes: portfolio strip, price chart, decision variables, risk controls, activity log
